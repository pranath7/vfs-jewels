## 🤖 Antigravity (AI Assistant)

That is a great question! 

Here is why we use **Test Mode** first:

1. **Production Mode blocks everything by default:** It sets `allow read, write: if false;`. Under this rule, when a customer visits your storefront and tries to place an order (or view their order tracking), Firebase will immediately block them because they are not logged in with an account.
2. **Test Mode allows guest checkout:** Test Mode sets a rule allowing anyone to read and write to the database temporarily (for 30 days). Since VFS Jewels allows **guest checkout** (customers don't need to register an account to buy, they just fill in their name and address), they need to write directly to the database.

### What we do for Production:
Once we confirm everything is working, we will write a custom set of rules on the **Rules** tab of Firestore that is safe for production:
* **Allow anyone to write orders** (so guest customers can place orders).
* **Allow anyone to lookup a specific order** if they have the correct `orderId` (for tracking).
* **Block anyone except the Admin** from seeing the list of all orders.

For now, selecting **"Start in test mode"** is the quickest way to get the system connected and tested!

service cloud.firestore {
